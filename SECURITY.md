# Security policy

## Reporting a vulnerability

Please **do not** open a public issue for undisclosed security problems in this sample application or its CI configuration.

- Use [GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) if enabled for this repository.
- Or contact the maintainers through a **private** channel with reproduction steps and impact.

For issues in the reusable **workflow-python** pipeline, use that repository’s **`SECURITY.md`** (e.g. **`thadiust/workflow-python`** on GitHub).
